# qrfile — File &#8596; QR Code Transfer Tool

Encode any file into one or more QR code images, and decode them back from a live camera feed or image files. **All processing is local — no data is ever transmitted externally.**

## Features

- **File &#8594; QR**: Compress and encode a file into QR code image(s). Large files are automatically split across multiple QR codes.
- **QR &#8594; File**: Decode QR code images back to the original file. Supports reading an entire directory at once, auto-detects multi-file mixed directories, and auto-groups parts by file.
- **Batch convert**: Recursively scan a directory and convert all text/code files to QR images at once.
- **Camera capture**: Live camera QR code scanning with automatic deduplication, part accumulation, and file reconstruction.

## Requirements

- Python 3.10+
- A camera (only required for the `camera` command)

## Installation

```bash
cd D:\MediaPipe\QRCode

# Install dependencies and register the `qrfile` command
pip install -e .
```

Dependencies: `zxing-cpp` (QR encode/decode), `opencv-python` (camera), `pillow` (image processing), `numpy`.

## Quick Start

### Encode: File &#8594; QR

```bash
# Encode a single file (auto-splits if large)
qrfile encode myfile.txt

# Specify output directory
qrfile encode myfile.txt --output ./my_qrcodes

# Omit text labels on QR images
qrfile encode myfile.txt --no-labels
```

### Decode: QR &#8594; File

```bash
# Pass a directory — all PNG images inside are read and decoded
qrfile decode ./my_qrcodes

# Restore to a specific directory (auto-detects multiple files)
qrfile decode ./my_qrcodes --output ./restored

# For a single file, --output can be a filename
qrfile decode ./my_qrcodes/hello_qrcode.png --output hello.txt

# Mix directories and individual files
qrfile decode ./my_qrcodes extra.png --output ./restored
```

When a directory contains QR codes for multiple files, they are **automatically grouped by file_id** and each file is restored independently.

> Compatible with the legacy `PART:X/Y:data` format generated by `file2qrcode.py`. No need to distinguish sources.

### Batch Convert: Directory &#8594; QR

```bash
# Convert all text/code files under ./docs
qrfile convert --source ./docs

# Custom output location
qrfile convert --source ./docs --output-base ./MyOutput

# Remove the 100-file limit
qrfile convert --source ./docs --max-files 0
```

Output goes to `<output-base>/<source-dirname>/`, e.g. `D:\MediaPipe\QRCode\OutputQR\docs\`.

**Supported file types** (recursive, matched by extension or full filename):

| Category | Extensions / Filenames |
|----------|----------------------|
| Code | `.py` `.c` `.cpp` `.h` `.hpp` `.cs` `.java` `.kt` `.js` `.ts` `.jsx` `.tsx` `.go` `.rs` `.rb` `.php` `.swift` `.lua` `.sql` `.vue` `.svelte` |
| Web | `.html` `.css` `.scss` `.sass` `.less` |
| Scripts | `.sh` `.bash` `.zsh` `.bat` `.cmd` `.ps1` |
| Config/Data | `.json` `.yaml` `.yml` `.toml` `.ini` `.cfg` `.csv` `.xml` `.md` `.rst` `.log` |
| Full names | `.env` `.env.example` `.env.local` `.gitignore` `.gitattributes` `.editorconfig` `Dockerfile` `Makefile` `CMakeLists.txt` `LICENSE` |

**Auto-skipped**: binary files (`.exe` `.dll` `.png` `.jpg` `.zip` `.pdf` etc.) and tool directories (`.git` `node_modules` `__pycache__` `.venv` etc.).

### Camera Capture

```bash
# Start live camera QR capture
qrfile camera

# Set initial resolution
qrfile camera --resolution 1280x720

# Specify output directory for restored files
qrfile camera --output ./captured_files

# Disable preview window
qrfile camera --no-preview
```

Keyboard controls during capture:

| Key | Action |
|-----|--------|
| `r` | Cycle through camera-supported resolutions |
| `q` | Quit |

**Status overlay** (top-left corner of the preview window, semi-transparent dark background):

- Green `[1] myfile.txt  14/14  DONE` — completed files (persistent)
- Yellow `[2] readme.txt  5/9` — in-progress parts

**Where are restored files saved?**

Current working directory by default. Use `--output` to specify a different directory (auto-created if missing).

**File naming**: `{file_id}_{sequence}{extension}`, e.g. `hello_001.txt`, `readme_002.txt`.

- **file_id**: derived from the original filename embedded in the QR codes (without extension)
- **sequence**: increments with each successfully restored file (`001`, `002`, `003`...)
- **extension**: auto-detected (text &#8594; `.txt`, PNG &#8594; `.png`, PDF &#8594; `.pdf`, unknown &#8594; `.bin`)

**Example**: capturing three files `notes.txt`, `data.json`, `logo.png` produces:

```
./notes_001.txt
./data_002.json
./logo_003.png
```

## How It Works

```
File bytes -> gzip compress -> base64 encode -> split -> PART header -> QR code image
```

QR codes use Version 40, error correction level L (`box_size=8`, `border=4`). Each QR code holds up to ~2900 bytes of base64 data.

**Data formats**:

- **Single file** (1 QR code): raw base64-encoded gzip data
- **Multi-file — new format** (generated by qrfile): `PART:<file_id>:<part>/<total>:<base64_chunk>`
- **Multi-file — legacy format** (generated by file2qrcode.py): `PART:<part>/<total>:<base64_chunk>`

Both formats are auto-detected during decoding. Parts are reassembled in order, base64-decoded, gzip-decompressed.

## Supported Resolutions

Resolutions are **auto-detected** at camera startup. The tool tries 12 common resolutions (from 3840&#215;2160 down to 320&#215;240) and keeps only those the camera actually accepts. Press `r` to cycle through the detected list. No manual configuration needed.

Typical detected results: `1920x1080`, `1280x720`, `640x480`, `640x360`.

# qrfile — 文件 &#8596; 二维码 传输工具

将任意文件压缩编码为一张或多张二维码图片，支持摄像头实时捕捉还原或从图片文件解码。**所有处理均在本地完成，不传输任何数据到外部。**

## 功能

- **文件 &#8594; QR 码**：将单个文件压缩编码为 QR 码图片（大文件自动分割为多张）
- **QR 码 &#8594; 文件**：将 QR 码图片解码还原为原始文件（支持目录批量读取、多文件混合自动分组）
- **批量转换**：递归扫描目录，批量转换所有文本/代码文件为 QR 码图片
- **摄像头捕捉**：摄像头实时捕捉 QR 码，自动去重、分片累积、还原文件

## 环境要求

- Python 3.10+
- 一个摄像头（仅 `camera` 命令需要）

## 安装

```bash
cd D:\MediaPipe\QRCode

pip install -e .
```

依赖包：`zxing-cpp`（QR 编解码）、`opencv-python`（摄像头）、`pillow`（图像处理）、`numpy`。

## 快速开始

### 编码：文件 &#8594; QR 码

```bash
qrfile encode myfile.txt
qrfile encode myfile.txt --output ./my_qrcodes
qrfile encode myfile.txt --no-labels
```

### 解码：QR 码 &#8594; 文件

```bash
# 传入目录，自动读取所有 PNG
qrfile decode ./my_qrcodes

# 还原到指定目录
qrfile decode ./my_qrcodes --output ./restored

# 单文件时可指定文件名
qrfile decode ./my_qrcodes/hello_qrcode.png --output hello.txt
```

目录中有多个文件的 QR 码时会**自动按 file_id 分组**，每个文件独立还原。

> 兼容 `file2qrcode.py` 生成的旧格式（`PART:X/Y:data` 无 file_id），无需区分来源。

### 批量转换：目录 &#8594; QR 码

```bash
qrfile convert --source ./docs
qrfile convert --source ./docs --output-base ./MyOutput
qrfile convert --source ./docs --max-files 0
```

输出路径：`<output-base>/<源目录名>/`。支持 `.py .c .cpp .js .ts .html .json .yaml .md .env` 等常见格式，自动跳过二进制文件和 `.git`/`node_modules` 等工具目录。

### 摄像头捕捉

```bash
qrfile camera
qrfile camera --resolution 1280x720
qrfile camera --output ./captured_files
```

| 按键 | 功能 |
|------|------|
| `r` | 在摄像头实际支持的分辨率间循环切换 |
| `q` | 退出 |

**预览窗口状态栏**（左上角半透明黑底）：

- 绿色 `[1] myfile.txt  14/14  DONE` — 已完成还原的文件（持续显示）
- 黄色 `[2] readme.txt  5/9` — 正在累积分片

**文件命名**：`{file_id}_{序号}{扩展名}`，如 `hello_001.txt`。序号按捕捉顺序递增，扩展名自动检测。

## 原理

```
文件字节 → gzip 压缩 → base64 编码 → 分片 → PART 头部 → QR 码图片
```

二维码采用 Version 40、纠错级别 L（`box_size=8`、`border=4`），单张最大约 2900 字节。

**数据格式**：
- 单文件：直接 base64 + gzip 编码
- 多文件（新格式）：`PART:<文件ID>:<序号>/<总数>:<base64分片>`
- 多文件（旧格式）：`PART:<序号>/<总数>:<base64分片>`

两种格式自动识别。

## 支持的分辨率

启动时**自动探测**摄像头实际支持的分辨率（从 3840&#215;2160 到 320&#215;240 共 12 种候选），仅保留硬件接受的选项。按 `r` 键在探测结果中循环切换。
